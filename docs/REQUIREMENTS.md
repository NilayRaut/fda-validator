> The shared reference. This is *what the system must do* and the *interfaces between components*. How we build it is in `BUILD_SPEC.md`; why we're building it is in `PROJECT_PROPOSAL.md`. If you change anything in §4 (interfaces), tell the whole team — it breaks other lanes.

## 1. Scope
A multi-agent system that, given a drug name and a question, produces a structured, evidence-grounded analysis that surfaces disagreement and confidence rather than a single recommendation. Drugs only (devices out of scope). Marketed/approved drugs are the realistic target since FAERS, Drugs@FDA, and the NDC directory all cover marketed products.

## 2. Components (each is a lane)
- **Planner** — decomposes the input into the two tracks' research questions.
- **Researcher: Safety & Efficacy** — clinical risk/benefit, grounded in FAERS + enforcement (web search for efficacy = optional/future, backend has none).
- **Researcher: Precedent & Market** — approval history + competitive landscape, grounded in Drugs@FDA + NDC directory (web search for market = optional/future, backend has none).
- **Adversarial critic** — independently seeks contradicting evidence; never reads the researchers' sources.
- **Disagreement engine** — assembles the ledger and computes confidence.
- **Synthesizer** — produces a report that presents conflict and evidence.
- **Observability (Weave)** — traces every step as an audit trail.
- **UI (Streamlit)** — input + rendered ledger + trace link.

## 3. Functional requirements

**FR-1 Planner.** Given `drug` + `question`, output exactly 2 research claims — one per track (safety+efficacy, precedent+market). Output must parse to the `claims` shape even if the model returns malformed JSON (defensive fallback required).

**FR-2 Parallel research.** The two researchers MUST run in parallel (runtime fan-out), not sequentially. Each MUST call its track's openFDA endpoint(s) and incorporate the returned data into its conclusion. Each produces one `Finding`.

**FR-3 Real grounding.** Every `Finding` MUST include `fda_data` populated from a live openFDA call and at least one item in `evidence` traceable to an FDA record (web `cited URL`s are optional/future since the backend has no web search). No fabricated citations.

**FR-4 Independent critic.** For each `Finding`, the critic MUST gather evidence *independently* (its own openFDA queries; adversarial web search is optional/future, backend has none) and MUST NOT be passed the researcher's `evidence`/`fda_data`. It outputs, per claim, a verdict in {`supports`, `contradicts`, `silent`} with its own sources.

**FR-5 Computed confidence.** Confidence MUST be computed from observable signals (source count, primary vs. secondary, critic verdict) by a deterministic function — NOT generated by an LLM. It produces a float 0–1 and a label in {`low`, `medium`, `high`}.

**FR-6 Conflict-preserving synthesis.** The synthesizer MUST present where the tracks/critic disagree and flag low-confidence claims. It MUST NOT collapse contested findings into a single confident verdict.

**FR-7 Audit trail.** Every node, every model call, every openFDA call, and the confidence computation MUST be traced in Weave, producing an inspectable tree for any run.

**FR-8 UI.** The UI MUST accept a drug name, run the pipeline, render the ledger (per-claim: positions, conflict, confidence badge, contradiction flag), and link to the run's Weave trace.

## 4. Interfaces (data contracts — the canonical shapes)

`Finding` (produced by each researcher):
| field | type | meaning |
|---|---|---|
| `stance` | str | `"safety_efficacy"` or `"precedent_market"` |
| `claim` | str | the question this track investigated |
| `conclusion` | str | the track's conclusion |
| `evidence` | list[dict] | each `{source, detail, url}` |
| `fda_data` | dict | raw openFDA payload used |

`DisagreementEntry` (produced by the disagreement engine):
| field | type | meaning |
|---|---|---|
| `topic` | str | what the entry is about |
| `positions` | list[dict] | each `{stance, conclusion, evidence}` |
| `conflict` | str | description of where they disagree |
| `confidence` | float | 0–1, computed |
| `confidence_label` | str | `low` / `medium` / `high` |

`ResearchState` (flows through the graph): `drug:str`, `question:str`, `claims:list[dict]`, `findings: list[Finding]` (append-only reducer — required), `contradictions:list[dict]`, `ledger:list[DisagreementEntry]`, `report:str`.

Critic output item (one per claim, appended to `contradictions`): `{stance, claim, verdict, evidence:list[dict]}` where `verdict ∈ {supports, contradicts, silent}`.

## 5. openFDA endpoint reference (no auth; ~240 req/min)
Base: `https://api.fda.gov/drug/<endpoint>.json?search=<query>&limit=<n>` (add `&count=<field>.exact` for aggregations).

| Endpoint | Track | Use | Example search field |
|---|---|---|---|
| `event` (FAERS) | Safety & Efficacy + critic | adverse-event signals | `patient.drug.medicinalproduct:"DRUG"` |
| `enforcement` | Safety & Efficacy + critic | recalls / enforcement | `product_description:"DRUG"` |
| `drugsfda` (Drugs@FDA) | Precedent & Market + critic | approval history, app type (NDA/ANDA/BLA), sponsor | `openfda.brand_name:"DRUG"` |
| `ndc` (NDC Directory) | Precedent & Market | marketed products, manufacturers, drug class | `openfda.generic_name:"DRUG"` or `pharm_class` |

Harmonized fields (`openfda.brand_name`, `openfda.generic_name`, `openfda.manufacturer_name`, `openfda.pharm_class_epc`) work across endpoints — use them to find the same drug consistently. `drug/drugshortages` is optional market enrichment (verify fields at build time).

## 6. Non-functional requirements
- **NFR-1 Resilience.** Every external call (W&B Inference LLM, openFDA) MUST fail soft — return a typed empty default and continue. The pipeline never crashes on a single failed call.
- **NFR-2 Latency/cost.** Agents use `deepseek-ai/DeepSeek-V3.1` via W&B Inference; `max_tokens` ≤ 1500; no web search on this backend. A full run should complete in well under ~2 minutes for demo.
- **NFR-3 Reproducibility.** Confidence is deterministic given the same findings + critic verdicts.
- **NFR-4 Observability.** A reviewer can open any run in Weave and see the full agent tree with inputs, outputs, latency, and the FDA data pulled.
- **NFR-5 No PII / safe data.** Only public openFDA data (and public web sources if web search is later added); no patient-identifying use.

## 7. Acceptance criteria (definition of done)
- Running on a single drug produces a `report` plus a `ledger` with ≥1 `DisagreementEntry`, each with a computed confidence and label.
- At least the two researchers run in parallel (visible as parallel branches in Weave).
- At least one `Finding` is grounded in a live openFDA payload, and the critic's verdict for at least one claim is backed by an independent source.
- The full run is traced end-to-end in Weave.
- Public GitHub repo; in-person demo.

## 8. Out of scope
Devices; drug *label*-based reasoning as a standalone component (approval-gated, dropped); generating actual FDA submission documents; any claim of regulatory/legal advice; fine-tuning or custom datasets.

## 9. Hard floor (ship-no-matter-what)
Planner + **2 parallel researchers** grounded in openFDA + independent critic + computed confidence + synthesizer, traced in Weave, public repo. Everything else is upside.
